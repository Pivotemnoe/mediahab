# Frontend Implementation Roadmap

## UI Phase 00 — Analysis and foundation
Route map, component map, design tokens, mock service layer, state model, responsive plan, and open questions. Do not build product pages yet.

## UI Phase 01 — Design system and shells
Global theme, primitives, marketing shell, auth shell, cabinet shell, desktop and mobile navigation, component showcase.

## UI Phase 02 — Landing, auth, dashboard
Marketing pages, pricing, authentication screens, dashboard with fixtures, responsive and required states.

## UI Phase 03 — Project Wizard
Project setup, AI rubric suggestion mock flow, platforms, tone, audience, example upload, confirmation.

## UI Phase 04 — Rubric Builder
Rubric CRUD UI, dynamic fields, repeatable groups, limits, rules, form preview, draft/version indicator.

## UI Phase 05 — Desktop Content Studio
Input blocks, voice mock, transcript review, master draft, AI suggestions, fact locks, platform previews, character budgets, autosave.

## UI Phase 06 — Mobile Voice PWA
Step flow, recording states, offline draft, resume, review and assemble, compact previews, installable PWA shell.

## UI Phase 07 — Integrations and Publications
Telegram/MAX/Instagram cards, connection states, capabilities, publication queue, partial success, retry/cancel, schedule modal.

## UI Phase 08 — Examples, Media, Calendar
Example library, import, filters, media library, calendar.

## UI Phase 09 — Billing, account, workspace
Pricing, current plan, usage, payment placeholder, invoices placeholder, roles, account settings.

## UI Phase 10 — Hardening
Real API contracts, accessibility, Playwright E2E, screenshot regression, performance, error boundaries, telemetry hooks, final responsive pass.

## Phase completion rule

Every phase ends with lint, tests, Playwright smoke tests, screenshots at 390/768/1280/1440 widths, changed-file list, and open questions. Do not proceed without approval.
