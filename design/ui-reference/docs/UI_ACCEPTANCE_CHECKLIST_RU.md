# Чек-лист приёмки UI

## Общие требования

- [ ] Название и логотип берутся из brand config.
- [ ] Нет хардкода конкретных рубрик в общих компонентах.
- [ ] Данные экранов поступают через service/repository layer или fixtures.
- [ ] Есть loading, empty, error и offline состояния.
- [ ] Нет горизонтального скролла на целевых ширинах.
- [ ] Основные действия доступны с клавиатуры.
- [ ] Focus видим.
- [ ] Формы имеют label, description и error.
- [ ] Опасные действия требуют подтверждения.

## Content Studio

- [ ] Отдельный счётчик символов для каждой площадки.
- [ ] Видно, какой текст master, а какой platform variant.
- [ ] AI-поля помечены.
- [ ] Факты пользователя можно заблокировать.
- [ ] Есть история или минимум индикатор несохранённых изменений.
- [ ] Черновик сохраняется автоматически.
- [ ] Media можно переставлять.
- [ ] Ошибка одной площадки не скрывает успешные публикации в других.

## Mobile PWA

- [ ] Основной сценарий проходится одной рукой.
- [ ] Запись имеет состояния idle/recording/paused/uploading/transcribing/error.
- [ ] После каждого блока есть подтверждение транскрипта.
- [ ] Можно вернуться к предыдущему блоку.
- [ ] Черновик восстанавливается после перезапуска.
- [ ] Есть понятное offline-состояние.

## Rubric Builder

- [ ] Можно добавить, удалить и переставить поле.
- [ ] Можно создать повторяемую группу.
- [ ] Можно задать источник значения.
- [ ] Можно задать обязательность.
- [ ] Можно задать редакционные лимиты.
- [ ] Технические лимиты площадки нельзя подменить пользовательским значением.
- [ ] Preview формы обновляется.

## Визуальная проверка

- [ ] Сняты Playwright screenshots на 390×844.
- [ ] Сняты Playwright screenshots на 768×1024.
- [ ] Сняты Playwright screenshots на 1280×800.
- [ ] Сняты Playwright screenshots на 1440×900.
- [ ] Реализация сравнена с выбранными референсами.
- [ ] Существенные отклонения описаны, а не скрыты.
