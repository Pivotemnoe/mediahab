# Название и бренд продукта

## Рекомендация на текущий этап

Использовать рабочее название **Temichev PostHub**.

В самом интерфейсе показывать коротко: **PostHub**.

Позиционирование:

> AI-конструктор контентных проектов и мультиплатформенных публикаций.

Короткий слоган:

> Создавайте один раз. Публикуйте везде.

Альтернативный слоган для главного экрана:

> Из диктовки — в готовый пост для каждой площадки.

## Почему не оставлять Temichev Media Hub как окончательное название

`Media Hub` часто воспринимается как библиотека файлов или DAM-система. Наш продукт шире: он создаёт структуру проекта, собирает материал, редактирует через AI, адаптирует и публикует.

## Почему не стоит сразу закреплять голое PostHub как окончательный бренд

Название понятное, но очень общее. В интернете уже существуют продукты и компании с названиями PostHub, ALL Post Hub, SmartPostHub и сходными вариантами. Поэтому PostHub подходит как рабочее имя и название категории, но перед коммерческим запуском потребуется отдельная проверка товарного знака, домена и поисковой уникальности.

## Как заложить переименование без переделки UI

Хранить бренд централизованно:

```ts
export const brand = {
  productName: "PostHub",
  fullName: "Temichev PostHub",
  tagline: "Создавайте один раз. Публикуйте везде.",
  supportEmail: "",
  logoMark: "PH"
};
```

Дополнительно предусмотреть:

```text
NEXT_PUBLIC_PRODUCT_NAME
NEXT_PUBLIC_PRODUCT_FULL_NAME
NEXT_PUBLIC_PRODUCT_TAGLINE
NEXT_PUBLIC_BRAND_LOGO_URL
```

Запрещено повторять название строкой в десятках компонентов.

## Варианты, которые можно рассмотреть позднее

1. **PostHub by Temichev** — понятное рабочее название для раннего запуска.
2. **Temichev PostHub** — лучше для первого собственного продукта и доверия аудитории.
3. Новый уникальный бренд — лучше для масштабирования, но только после проверки товарного знака и доменов.

На этапе разработки не нужно тормозить проект из-за нейминга. Архитектура должна позволить заменить бренд за один коммит.
