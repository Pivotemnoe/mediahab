# Направление продукта и UI

## Главная идея

PostHub — не форма кросспостинга. Это операционная система контента, где пользователь сначала описывает собственный проект и рубрики, затем создаёт материал в привычной форме, а система помогает довести его до публикации.

## Основной пользовательский результат

Пользователь должен за несколько минут пройти путь:

```text
выбрал проект
→ выбрал рубрику
→ надиктовал факты и впечатления
→ проверил распознавание
→ получил крючок, оценки и собранный текст
→ увидел версии Telegram / MAX / Instagram
→ отредактировал
→ опубликовал
```

## Приоритеты интерфейса

1. Быстрый вход в создание контента.
2. Пошаговая диктовка без необходимости придумывать структуру на ходу.
3. Сохранение фактов и прозрачность AI-изменений.
4. Реальный предпросмотр каждой площадки.
5. Конструктор проектов и рубрик без программиста.
6. Понятные статусы публикации и ошибок.
7. Мобильная работа как основной сценарий, а не урезанная копия desktop.

## Финальная комбинация референсов

### Desktop

Взять из Editorial Studio:

- светлый редакционный стиль;
- левую навигацию;
- проектный контекст;
- центральный редактор;
- правую панель предпросмотра;
- ясную иерархию действий.

### Конструктор

Взять из Visual Builder:

- блоки полей;
- drag-and-drop порядок;
- настройки поля справа;
- лимиты, обязательность и источник значения;
- preview создаваемой формы.

### Mobile PWA

Взять из Mobile-first PWA:

- крупную кнопку записи;
- пошаговый прогресс;
- один вопрос на экран;
- быстрые варианты «добавить блюдо», «пропустить», «готово»;
- сохранение черновика после каждого блока;
- экран проверки перед сборкой.

### Operations

Из Command Center позднее использовать:

- тёмную тему;
- календарь;
- очередь публикаций;
- статусы интеграций;
- расширенную аналитику.

## Визуальный характер

Интерфейс должен быть:

- спокойным;
- редакционным;
- современным;
- тёплым, но не «ресторанным»;
- профессиональным, но не корпоративно-холодным;
- насыщенным функциями без визуального шума.

## Главные UX-правила

- На одном экране только одно главное действие.
- Важные факты пользователя можно заблокировать от изменения AI.
- AI-предложения всегда отмечены визуально.
- Любая генерация имеет «до/после» или историю версии.
- Публикация всегда требует явного подтверждения.
- Для каждой площадки показан отдельный счётчик символов.
- Ошибка одной площадки не блокирует успех остальных.
- Черновик сохраняется автоматически.
