# UI Product Direction

## Product definition

PostHub is not a simple cross-posting form. It is a content operations workspace where users define their own projects and rubrics, capture source material, use AI as an editor, adapt content per platform, and publish with control.

## Primary user outcome

The user should be able to complete this flow quickly:

```text
Select project
→ Select rubric
→ Record facts and impressions
→ Review transcription
→ Generate hook, suggested ratings, and assembled draft
→ Review Telegram / MAX / Instagram variants
→ Edit
→ Publish
```

## Reference combination

### Editorial Studio
Use as the primary desktop shell, dashboard, content editor, and platform preview layout.

### Visual Builder
Use for project builder, rubric schema builder, dynamic fields, rules, examples, and limits.

### Mobile-first PWA
Use for voice capture, step-by-step field completion, draft review, and publish approval.

### Command Center
Reserve for later dark mode, calendar, publication operations, platform health, and analytics.

## UX principles

- One primary action per screen.
- User facts can be locked against AI edits.
- AI-generated or AI-modified values are visibly marked.
- Autosave after each meaningful step.
- Separate preview and character budget per platform.
- Explicit confirmation before live publishing.
- A failure on one platform does not invalidate other publications.
- Mobile is a first-class workflow.
