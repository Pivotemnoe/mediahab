# Frontend и UX

## Цели

Mobile-first для диктовки и медиа с телефона; быстрый переход к следующему блоку; явное различие между фактами, предложениями AI, правками пользователя и опубликованным текстом; отдельный предпросмотр каждой платформы; простые настройки по умолчанию и расширенный режим для экспертов.

## Лендинг

Hero, проблема и результат, диктовка, конструктор проектов и рубрик, AI-редактор и примеры, Telegram/MAX/Instagram, сценарии использования, тарифы, безопасность, регистрация.

## Навигация кабинета

Dashboard, Создать, Проекты, Контент, Публикации, Медиа, Настройки. На мобильном — нижняя навигация.

## Конструктор проекта

Пошаговый мастер с автосохранением. AI-рубрики показываются карточками «Принять», «Изменить», «Удалить». Можно менять порядок и архивировать. JSON по умолчанию скрыт.

## Конструктор рубрики

Палитра полей, canvas формы и инспектор настроек. Добавление обычных и повторяемых блоков, источник user/AI, обязательность, лимиты, генерируемые поля, мобильный preview, сохранение новой версии.

## Контент-студия

- Заголовок: проект, рубрика, статус, автосохранение, диапазон знаков, «Собрать».
- Пошаговый режим: подсказка, запись, текст, назад/далее, «Добавить блюдо».
- Полный режим: все блоки, происхождение, lock, transcript и медиа.
- AI-review: сравнение исходника и результата, подсветка конфликтов, варианты крючка, редактируемые оценки, регенерация отдельного раздела.

Нельзя оставлять только кнопку «Перегенерировать всё».

## Медиа

Множественная загрузка, порядок drag-and-drop, тип и длительность, предупреждения по площадкам, выбор обложки, прогресс. Удаление из поста не обязано сразу удалять файл из хранилища.

## Предпросмотр

Вкладки Telegram, MAX, Instagram. Показываются конечный аккаунт, текст, два счётчика, layout медиа, ошибки, редактирование, утверждение и публикация. Для Telegram явно показывать Rich Message или fallback с разделением.

## Публикация

Одно окно подтверждения со всеми площадками и независимыми статусами. Частичный успех сохраняется. Для ошибки: повтор, правка версии, детали.

## Биллинг

Тариф, расход, тарифная таблица и кнопка оплаты. В MVP честно показывать «в разработке» или ручное подключение, а не имитировать успешную оплату.

## PWA и доступность

Установка, app shell, локальный кэш черновиков, уведомление об обновлении service worker, offline badge. AI и публикация недоступны офлайн с объяснением. Семантические компоненты, keyboard navigation, focus management и доступные ошибки.
