# Критерии приёмки

## Общие

Чистый checkout запускается по документации; миграции работают; рубрики и лимиты не хардкодятся; чужой workspace недоступен; entitlements проверяются backend; клиент генерируется из OpenAPI; секретов нет в коде, логах и browser storage; все проверки проходят.

## Конструктор

Владелец через UI создаёт проект, добавляет и меняет рубрики, порядок, архив, пользовательские повторяемые блоки и лимиты. Сохранение создаёт новую версию, старые материалы не ломаются. Пресет «Что поесть? Армавир» импортируется идемпотентно.

## Студия

«Обзор недели» спрашивает настроенные фактические блоки, но не крючок и оценки. Блюда повторяемы. Голос идёт напрямую в S3, распознаётся, исправляется и блокируется. Reload не теряет черновик. Изменение AI зафиксированного заведения, адреса, чека, блюда или цены блокирует результат.

## AI

Провайдеры меняются настройкой; JSON валидируется; выбирается 3–8 примеров; крючок и оценки помечены как предложения; пользовательские оценки приоритетны; editor mode сохраняет факты; каждый run имеет модель, версии, usage и результат.

## Telegram

Fixture: 4069 знаков, 10 медиа — 7 фото и 3 видео. Выбран Rich Message. Payload в пределах 32768/50. При наличии ключей создаётся одна видимая публикация с коллажем и текстом, сохраняется ID, повтор не создаёт дубль. Fallback требует подтверждения.

## MAX

Больше 4000 блокируется или адаптируется; `attachment.not.ready` повторяется; токен не идёт в query; webhook дедуплицируется; тест количества смешанных медиа записывается как live evidence.

## Instagram

Отдельная версия до 2200; карусель 2–10; quota проверяется; container polling; недостаток прав даёт понятный `manual_required`; retry не дублирует опубликованный container.

## Публикации

Частичный успех виден; перезапуск worker не теряет и не дублирует задачу; timezone и отмена работают; история повторов доступна.

## SaaS и биллинг

Есть лендинг, регистрация, вход, pricing, кабинет, account и billing. Пользователь видит только свои данные. Free limit проверяется backend. Mock payment не выглядит как настоящая успешная оплата.

## Эксплуатация

Health/readiness работают, backup/restore выполнимы, staging по умолчанию не может публиковать в production-канал.
