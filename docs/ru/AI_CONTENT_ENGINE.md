# AI-движок контента

## Цель

Сохранять факты и голос пользователя, автоматизируя структуру, крючки, оценки, переходы, адаптацию и проверку. Это не общий чат «напиши что-нибудь», а контролируемый конвейер с версиями и структурированными результатами.

## Провайдеры

Отдельные интерфейсы `TextGenerationProvider`, `SpeechToTextProvider` и `EmbeddingProvider`. В бизнес-модулях нельзя вызывать конкретный API напрямую. Модели и fallback настраиваются.

## Манифест контекста

Каждая генерация сохраняет точные ID версий проекта, рубрики, правил, промптов, примеров, hash зафиксированных фактов и версию платформенной политики. Благодаря этому результат можно воспроизвести и проверить.

## Задачи

- **Извлечение фактов:** структурированный JSON по схеме рубрики, источник каждого значения и список неуверенности.
- **Сборка master:** блоки текста, несколько вариантов крючка, предложение оценок, CTA, карта использованных фактов и предупреждения.
- **Оценки:** только предложение с кратким обоснованием из фактов; пользовательская оценка не меняется.
- **Адаптация:** отдельная редакция по стратегии `preserve_full`, `compress`, `micro_post` или `metadata_package`.
- **Quality check:** структурированные ошибки, предупреждения и оценки. Длина, обязательные поля и равенство фактов проверяются детерминированным кодом, а не мнением модели.

## Слои промпта

1. Системная роль и JSON-схема.
2. Запрет выдумывать и правило fact lock.
3. Стиль проекта.
4. Структура рубрики и длина.
5. Утверждённые примеры.
6. Запрещённые паттерны.
7. Факты текущего материала.
8. Текущая задача.

Примеры явно помечаются как образцы стиля, а не источник фактов.

## Retrieval

Сначала фильтрация по workspace и проекту, затем vector search. Предпочитать ту же рубрику, учитывать ручную оценку и engagement. Сохранять scores. Отклонённые примеры не использовать как положительные.

## Регрессионные тесты

Для каждого проекта хранить наборы: исходные блоки, locked facts, обязательные секции, разрешённые факты, запрещённые фразы, диапазон длины и утверждённый ориентир. Сравнивать не точные слова, а факты, структуру, длину, шкалу оценок и стиль. Первый кейс — «У Доника».

## Ошибки

- Timeout: повтор, затем резервный провайдер.
- Невалидная схема: одна попытка repair, затем fallback или явная ошибка.
- Конфликт фактов: блокировать, не исправлять молча.
- Не попали в длину: один целевой проход сокращения/расширения, затем предупреждение.
- Недоступность провайдера: health-state и fallback.

## Настройки пользователя

AI-режим, сила редактирования, юмор, провайдер и модель, число крючков, автопредложение оценок, retrieval и число примеров. Сырой prompt editor — только в экспертном режиме.
