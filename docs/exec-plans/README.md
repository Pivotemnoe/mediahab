# Execution plans

Codex creates one plan per implementation phase or substantial subtask. Name files like:

```text
2026-06-20-phase-00-platform-spikes.md
2026-06-25-phase-01-foundation.md
```

Each plan must contain:

1. Objective and non-goals.
2. Current-state findings.
3. Assumptions and unresolved questions.
4. Files/modules to create or change.
5. Database migrations and rollback notes.
6. Security and tenancy impact.
7. External API/live-test prerequisites.
8. Step-by-step implementation order.
9. Unit, integration, contract, connector, and E2E tests.
10. Commands to run.
11. Demo/acceptance evidence.
12. Risks and recovery strategy.
13. Status/progress notes updated during execution.
