# Как передать этот пакет Codex

1. Распакуйте архив в пустую рабочую папку.
2. До разработки создайте Git-репозиторий, чтобы каждое изменение Codex можно было проверить и откатить.
3. Откройте в Codex корень репозитория. Корневой `AGENTS.md` должен загрузиться автоматически.
4. Первым заданием передайте содержимое `codex/00_START_HERE_EN.md`.
5. Поручите выполнить **только этап 00**. Не просите Codex собрать весь продукт за один запуск.
6. Проверьте ADR, список открытых вопросов, результаты платформенных spike-тестов и предлагаемые изменения ТЗ.
7. Утверждайте по одному этапу. На каждом этапе требуйте миграции, тесты, точные команды и результаты, обновление документации и проверку diff.
8. Реальные ключи передавайте только через переменные окружения или хранилище секретов. Не вставляйте production-секреты в Markdown и код.
9. Английские документы остаются каноническими. При изменении поведения обновляйте русскую версию.
10. После изменения YAML/JSON-схем и пресетов повторно запускайте валидацию репозитория.

Рекомендуемая команда после утверждения этапа 00:

> Implement Phase 01 only. First write an execution plan under `docs/exec-plans/`, list assumptions and risks, then implement the smallest runnable vertical foundation. Run every required check from `AGENTS.md`, review the diff, and report exact results. Do not begin Phase 02.
