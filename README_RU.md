# Temichev Media Hub — пакет технического задания для Codex

Это готовый пакет для передачи Codex: мультитенантный SaaS-конструктор контента, который принимает текст или диктовку, собирает проверяемый мастер-пост, создаёт отдельные версии под площадки и публикует их в Telegram, MAX, Instagram и будущие каналы.

## С чего начинать

1. Открыть `AGENTS.md`.
2. Открыть `docs/en/MASTER_SPEC.md` — это каноническая версия для Codex.
3. Для проверки на русском открыть `docs/ru/MASTER_SPEC.md`.
4. Передать Codex текст из `codex/00_START_HERE_EN.md`.
5. Сначала выполнить только `plans/PHASE_00_DISCOVERY_AND_SPIKES.md`.

## Главный принцип

Первый настроенный проект — **«Что поесть? Армавир»**, но логика приложения не должна быть зашита под этот канал. Пользователь через фронтенд сам создаёт проекты, рубрики, поля диктовки, правила, промпты, примеры, лимиты и подключения.

## Важная граница

Через интерфейс можно без программиста создать ручную площадку или generic webhook. Но полноценная автоматическая интеграция с новой социальной сетью требует отдельного backend-коннектора: у каждой платформы свои OAuth, токены, загрузка медиа, лимиты, ошибки, редактирование и удаление.

## Что находится в пакете

- `AGENTS.md` с постоянными правилами для Codex.
- Полное ТЗ на английском и русском языках.
- Дерево будущего репозитория, модель данных, API, AI-конвейер, UX, безопасность, биллинг, инфраструктура, тестирование и план разработки по этапам.
- Машиночитаемые правила платформ и пресет «Что поесть? Армавир».
- JSON Schema для ключевых сущностей конструктора.
- Готовые поэтапные задания для Codex.
- Реальный Telegram-регрессионный пример: 4069 знаков и 10 смешанных медиафайлов в одной публикации.
- Список официальной документации, проверенный 20 июня 2026 года.
